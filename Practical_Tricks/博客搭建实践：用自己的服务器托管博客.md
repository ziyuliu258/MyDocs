### 一、 DNS 解析：分线路流量分流 (DNSPod)
**操作目标**：实现国内流量直连香港服务器，国外流量走 GitHub Pages。
1. **修改域名 NS 地址**：在域名注册商处将 DNS 服务器修改为 `anchovy.dnspod.net` 和 `pakchoi.dnspod.net`。
2. **配置解析记录**：在 DNSPod 控制台手动添加以下记录：

|**主机记录**|**记录类型**|**线路类型**|**记录值**|
|---|---|---|---|
|`@`|`A`|**默认**|`38.38.251.209`|
|`www`|`A`|**默认**|`38.38.251.209`|
|`@`|`CNAME`|**境外**|`ziyuliu258.github.io`|
|`www`|`CNAME`|**境外**|`ziyuliu258.github.io`|

> **注意**：必须先创建“默认”线路，才能创建“境外”线路。如果有残留的 `AAAA` 记录（IPv6）必须删除，否则会干扰后续的 SSL 申请。

### 二、 服务器端环境配置 (HK Server)
**操作目标**：搭建 Nginx 服务并配置 Git 自动化部署钩子。
#### 1. 安装基础服务
```bash
sudo apt update && sudo apt install nginx git -y  # 假设为 Debian/Ubuntu 环境
```

#### 2. 规范化路径迁移 (解决 Permission Denied)
由于 Nginx 无法访问 `/root` 目录，必须将博客根目录设在标准路径：

```bash
sudo mkdir -p /var/www/blog/public # 创建 Web 根目录
sudo chown -R www-data:www-data /var/www/blog # 授权给 Nginx 用户
sudo chmod -R 755 /var/www/blog # 确保目录可执行
```

#### 3. 配置 Git 自动化钩子 (Git Hooks)
在服务器创建裸仓库以接收代码：
```bash
git init --bare /root/blog.git # 创建 Git 裸仓库
nano /root/blog.git/hooks/post-receive # 编辑钩子脚本
```
在脚本中写入以下内容：
```bash
#!/bin/bash
# 自动将代码检出到指定的 Web 目录
git --work-tree=/var/www/blog/public --git-dir=/root/blog.git checkout -f
echo "Success: Blog deployed to HK server!"
```
赋予执行权限：
```bash
chmod +x /root/blog.git/hooks/post-receive
```
#### 4. 配置 Nginx 虚拟主机
创建配置文件 `/etc/nginx/conf.d/blog.conf`：
```nginx
server {
    listen 80;
    server_name ziyuliu258.cn www.ziyuliu258.cn;
    root /var/www/blog/public;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

重启服务：
```bash
sudo nginx -t && sudo systemctl restart nginx
```

### 三、 本地 Hexo 配置与部署 (Arch/EndeavourOS)
**操作目标**：实现一键双向推送。
1. **配置免密登录**：
```bash
ssh-copy-id root@38.38.251.209 # 确保本地执行 hexo d 时无需输入密码
```
2. **修改 `_config.yml`**：
```yaml
deploy:
  type: git
  repo:
    github: git@github.com:ziyuliu258/ziyuliu258.github.io.git,master
    hk: root@38.38.251.209:/root/blog.git,master
```
3. **执行推送**：
```bash
hexo clean && hexo g -d
```

### 四、 SSL/HTTPS 证书部署 (Certbot)
**操作目标**：解决 `PR_END_OF_FILE_ERROR`，实现全站加密。
1. **临时修改解析记录**：由于 Let's Encrypt 验证服务器在境外，需临时将 DNSPod 中的“境外”线路也改为指向香港 IP (`38.38.251.209`)。
2. **申请证书**：

```bash
sudo apt install python3-certbot-nginx -y
sudo certbot --nginx -d ziyuliu258.cn -d www.ziyuliu258.cn
```

3. **恢复分流**：证书申请成功后，将 DNSPod 境外线路改回 `CNAME` 指向 GitHub。

### 五、 常用诊断调试命令
如果你在本地遇到无法连接的问题，请使用以下“组合拳”定位：
- **本地 DNS 强制重定向**（修改 `/etc/hosts`）：
    `38.38.251.209 ziyuliu258.cn`
- **查看解析 IP**：
    `nslookup ziyuliu258.cn`
- **查看 HTTP 响应头**：
    `curl -I http://ziyuliu258.cn`
- **查看服务器权限报错**：
    `tail -n 20 /var/log/nginx/error.log`
