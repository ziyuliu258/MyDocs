## 一、概论

### 1.1 计算机系统组成
- **硬件系统**：运算器、控制器、存储器、输入设备、输出设备
- **软件系统**：系统软件、应用软件

### 1.2 计算机硬件架构

#### 冯·诺伊曼架构
- **五大部件**：运算器、控制器、存储器、输入、输出
- **特点**：存储程序、顺序执行
- **缺点**：I/O需经运算器，浪费运算时间

#### 现代计算机结构
- **核心特点**：以存储器为中心
- **CPU组成**：运算器 + 控制器

### 1.3 计算机系统层次结构
- 软件和硬件的**逻辑功能等价性**
- 层次：硬件层 → 微程序层 → 机器语言层 → 操作系统层 → 高级语言层

### 1.4 基本工作原理

#### 运算器组成
- **ACC**（累加器）、**ALU**（算术逻辑单元）、**MQ**（乘商寄存器）、**X**（操作数寄存器）

#### 主存储器组成
- **存储体**、**MAR**（存储器地址寄存器）、**MDR**（存储器数据寄存器）

#### 控制器工作流程
1. **取指**：`(PC)→MAR` → 访存 → `(MDR)→IR`
2. **分析**：`OP(IR)→CU`
3. **执行**：`Ad(IR)→MAR` → 取操作数 → 执行

### 1.5 ⚠️ 性能指标（重点公式）

| 指标 | 定义/公式 |
|------|----------|
| 机器字长 | = ALU位数 = 通用寄存器宽度 |
| 主存容量 | = 存储单元数 × 存储字长 |
| CPI | 执行一条指令的时钟周期数 |
| CPU执行时间 | = 指令数 × CPI × 时钟周期 |
| IPC | = 1/CPI |
| MIPS | = (f/CPI) ÷ 10⁶ |
| MFLOPS | 每秒百万次浮点运算 |

### 1.6 ⚠️ 各种"字长"概念辨析

#### 核心概念定义

| 字长类型 | 定义 | 决定因素 |
|---------|------|----------|
| **机器字长** | CPU一次能处理的二进制位数 | = ALU位数 = 通用寄存器宽度 |
| **存储字长** | 主存一个存储单元的二进制位数 | 存储器设计 |
| **指令字长** | 一条指令包含的二进制位数 | 指令系统设计 |
| **数据字长** | 数据总线一次能传输的位数 | = 数据总线宽度 |
| **地址字长** | 地址总线的位数 | = 地址总线宽度，决定寻址范围 |

#### 关系图解
```
┌─────────────────────────────────────────────────────────┐
│                         CPU                              │
│  ┌──────────────────────────────────────────────────┐   │
│  │  机器字长 = ALU位数 = 通用寄存器宽度              │   │
│  │  (如32位/64位CPU)                                │   │
│  └──────────────────────────────────────────────────┘   │
└───────────────┬─────────────────────┬───────────────────┘
                │                     │
         数据总线宽度            地址总线宽度
         (数据字长)              (地址字长)
                │                     │
                ↓                     ↓
┌───────────────────────────────────────────────────────┐
│                      主存储器                          │
│  ┌─────────────────────────────────────────────────┐  │
│  │  存储字长 = 一个存储单元的位数                   │  │
│  │  寻址范围 = 2^(地址总线宽度)                     │  │
│  └─────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────┘
```

#### ⚠️ 常见关系与区别

| 关系             | 说明                     |
| -------------- | ---------------------- |
| 机器字长 = 数据总线宽度  | **通常相等**（现代计算机），但不绝对   |
| 机器字长 ≠ 存储字长    | 可能不等，按字节编址时存储字长=8位     |
| 指令字长           | 可以是机器字长的整数倍（定长）或可变（变长） |
| 地址总线宽度         | 与机器字长**无必然关系**，由寻址需求决定 |
| MDR宽度 = 数据总线宽度 | MDR需要与数据总线匹配           |
| MAR宽度 = 地址总线宽度 | MAR需要能表示所有地址           |

#### 典型例题分析

**例**：某计算机字长32位，按字节编址，主存4GB，求各字长？

- **机器字长** = 32位
- **数据总线宽度** = 32位（通常等于机器字长）
- **地址总线宽度** = $\log_2(4GB / 1B)$ = $\log_2(2^{32})$ = **32位**
- **存储字长** = 8位（按字节编址）
- **MAR宽度** = 32位（能寻址4GB）
- **MDR宽度** = 32位（与数据总线匹配）

**注意**：若要求"指令按字对齐"，则PC每次+4（而非+1）

#### 易混淆点总结

1. **机器字长 vs 存储字长**
   - 机器字长：CPU处理能力
   - 存储字长：存储单元大小
   - 按字编址时两者相等；按字节编址时存储字长=8位

2. **数据总线 vs 地址总线**
   - 数据总线：传数据，双向，宽度影响传输效率
   - 地址总线：传地址，单向，宽度决定寻址空间

3. **指令字长的变化**
   - RISC：定长，通常 = 机器字长
   - CISC：变长，可以是机器字长的1/2、1、2倍等

## 二、数据表示

### 2.1 定点数

#### 原码/反码/补码/移码
- **原码**：符号位 + 真值绝对值
- **反码**：正数同原码；负数数值位取反
- **补码**：正数同原码；负数 = 反码 + 1
- **移码**：$[x]_移 = x + 2^n$，**仅用于定点整数**

#### ⚠️ 重要特性
- 补码和移码对0只有**一种表示**
- 8位补码 `10000000` 表示 `-128`（位权法）
- **扫描法**：右起第一个1及其右边不变，其余取反

### 2.2 浮点数

#### 一般格式
$$N = M \times r^E$$
- M：尾数（决定精度）
- E：阶码（决定范围）
- r：基数（隐含）

#### 浮点数规格化
- **定义**：尾数原码最高数值位必须为1
- **目的**：表示形式唯一，精度最高

#### ⚠️ IEEE 754 标准（重点）

| 类型 | 符号 | 阶码 | 尾数 | 偏置常数 |
|------|------|------|------|----------|
| float (32位) | 1位 | 8位 | 23位 | 127 |
| double (64位) | 1位 | 11位 | 52位 | 1023 |

- **阶码**：移码形式，但偏置常数为 $2^{k-1}-1$
- **尾数**：原码形式，隐藏最高位1，实际为 `1.M`

### 2.3 数据类型转换
- **int → float**：高8位非0时**精度丢失**
- **float → int**：小数截断，可能溢出
- **int → double**：精确表示

## 三、运算方法

### 3.1 移位运算

| 类型 | 对象 | 规则 |
|------|------|------|
| 逻辑移位 | 无符号数 | 补0 |
| 算术移位 | 有符号数 | 正数补0；负数原码补0，反码补1，补码左移补0右移补1 |
| 循环移位 | - | 带/不带进位标志CF |

### 3.2 ⚠️ 溢出检测（三种方法）
1. **符号位比较法**：同号相加，结果异号则溢出
2. **进位比较法**：最高数值位进位 ⊕ 符号位进位 = 1 则溢出
3. **变形补码法**：双符号位，`01`正溢，`10`负溢

### 3.3 加法器硬件实现

#### 串行进位加法器
- 结构简单，速度慢
- 延迟：$n \times T_{FA}$

#### 先行进位加法器（CLA）
- **进位生成函数**：$G_i = A_i \cdot B_i$
- **进位传递函数**：$P_i = A_i \oplus B_i$
- 并行计算进位，速度快

### 3.4 乘法运算

#### 原码一位乘法
- **符号单独处理**：积的符号 = 被乘数符号 ⊕ 乘数符号
- **数值部分相乘**：采用"加法+移位"实现

#### ⚠️ Booth算法（补码一位乘法）

##### 算法原理
Booth算法直接对**补码**进行运算，符号位参与运算，无需单独处理。

##### 核心思想
利用乘数相邻两位的差值来决定操作：
- 设乘数为 $Y = y_0.y_1y_2...y_n$（$y_0$ 为符号位）
- 引入辅助位 $y_{n+1} = 0$
- 根据 $y_i y_{i+1}$ 的组合决定操作

##### 操作规则

| $y_i$ | $y_{i+1}$ | $y_{i+1} - y_i$ | 操作 |
|-------|-----------|-----------------|------|
| 0 | 0 | 0 | 部分积右移一位 |
| 0 | 1 | +1 | 部分积 **加** $[X]_补$，然后右移一位 |
| 1 | 0 | -1 | 部分积 **加** $[-X]_补$，然后右移一位 |
| 1 | 1 | 0 | 部分积右移一位 |

##### 运算流程
1. **初始化**：
   - 部分积 P = 0
   - 被乘数 X 存入 X 寄存器
   - 乘数 Y 存入 MQ 寄存器，末尾添加辅助位 $y_{n+1} = 0$
   
2. **迭代 n+1 次**（n 为数值位数）：
   - 检查 MQ 最低两位 $y_i y_{i+1}$
   - 根据操作规则执行加法或减法
   - **算术右移**一位（符号位保持不变）
   
3. **最后一步不移位**，直接得到结果

##### 计算示例
设 $X = -0.1101$，$Y = +0.1011$，求 $X \times Y$

**准备工作**：
- $[X]_补 = 1.0011$
- $[-X]_补 = 0.1101$
- $[Y]_补 = 0.1011$

**运算过程**（采用双符号位）：

| 步骤 | 部分积P | 乘数MQ | 辅助位 | 操作说明 |
|------|---------|--------|--------|----------|
| 初始 | 00.0000 | 1011 | 0 | $y_4y_5 = 10$，加 $[-X]_补$ |
| 第1步 | 00.1101 | 1011 | 0 | 加 $[-X]_补$ 后右移 |
|  | 00.0110 | 1101 | 1 | 右移结果，$y_3y_4 = 11$ |
| 第2步 | 00.0110 | 1101 | 1 | 仅右移 |
|  | 00.0011 | 0110 | 1 | 右移结果，$y_2y_3 = 01$ |
| 第3步 | 00.0011 | 0110 | 1 | 加 $[X]_补$ |
|  | 11.0110 | 0110 | 1 | 加后结果 |
|  | 11.1011 | 0011 | 0 | 右移结果，$y_1y_2 = 10$ |
| 第4步 | 00.1000 | 0011 | 0 | 加 $[-X]_补$ |
|  | 00.0100 | 0001 | 1 | 右移结果，$y_0y_1 = 01$ |
| 第5步 | 00.0100 | 0001 | 1 | 加 $[X]_补$，**不移位** |
|  | 11.0111 | 0001 | 1 | 最终结果 |

**结果**：$[X \times Y]_补 = 1.01110001$，真值为 $-0.10001111$

##### 算法特点
1. **符号位参与运算**：无需单独处理符号
2. **采用算术右移**：保持符号位不变
3. **迭代次数**：n+1 次（比原码乘法多一次）
4. **最后一步不移位**
5. **适用于定点整数和定点小数**

##### 与原码乘法对比

| 特性 | 原码一位乘法 | Booth算法 |
|------|-------------|-----------|
| 符号处理 | 单独异或 | 参与运算 |
| 操作数形式 | 原码 | 补码 |
| 迭代次数 | n 次 | n+1 次 |
| 移位方式 | 逻辑右移 | 算术右移 |
| 最后一步 | 移位 | 不移位 |

### 3.5 ⚠️ 除法运算

#### 原码除法

##### 恢复余数法
- **基本思想**：余数减除数，若结果<0则恢复（加回除数）
- **操作次数**：不固定，最多2n次加减

##### 加减交替法（不恢复余数法）
- **核心规则**：
  - 余数≥0：商1，左移，**减**除数
  - 余数<0：商0，左移，**加**除数
- **操作次数**：固定n+1次

##### 运算流程
```
初始：ACC←|X|, MQ←0, X寄存器←|Y|
第一步：ACC ← ACC - X
循环n次：
  若ACC≥0：MQ末位←1，(ACC,MQ)左移，ACC←ACC-X
  若ACC<0：MQ末位←0，(ACC,MQ)左移，ACC←ACC+X
最后：若ACC<0，需恢复余数
符号：商符 = 被除数符 ⊕ 除数符
```

#### 补码除法（加减交替法）
- **核心规则**：
  - 余数与除数**同号**：商1，左移，**减**除数
  - 余数与除数**异号**：商0，左移，**加**除数
- **商的校正**：末位恒置1

#### 除法对比

| 算法 | 符号处理 | 核心判断 | 操作次数 |
|------|---------|---------|---------|
| 恢复余数法 | 单独处理 | 余数符号 | ≤2n |
| 原码加减交替 | 单独处理 | 余数符号 | n+1 |
| 补码加减交替 | 参与运算 | 余数与除数符号关系 | n+1 |

### 3.6 ⚠️⚠️ 浮点数运算（重点）

#### 浮点数加减法步骤
1. **对阶**：小阶向大阶看齐（尾数右移，阶码增大）
2. **尾数加减**：补码运算
3. **规格化**：
   - 左规：尾数左移，阶码-1（结果过小）
   - 右规：尾数右移，阶码+1（尾数溢出）
4. **舍入处理**
5. **溢出判断**：检查阶码是否上溢/下溢

#### ⚠️ 对阶原则
**小阶向大阶对齐**，因为：
- 大阶向小阶：尾数左移，**高位有效数字丢失**，误差大
- 小阶向大阶：尾数右移，**低位丢失**，误差小

#### 浮点数乘法
$$X \times Y = (M_X \times M_Y) \times 2^{E_X + E_Y}$$
- 阶码**相加**
- 尾数**相乘**（定点乘法）
- 最多**左规1次**

#### 浮点数除法
$$X \div Y = (M_X \div M_Y) \times 2^{E_X - E_Y}$$
- 阶码**相减**
- 尾数**相除**（要求 $|M_X| < |M_Y|$）
- 最多**右规1次**

#### 浮点运算溢出
- **上溢**：阶码超过最大值，机器停止运算
- **下溢**：阶码低于最小值，按机器零处理

### 3.7 ⚠️ 浮点数舍入方法

#### 舍入原因
1. 对阶时尾数右移，低位被移出
2. 尾数乘法结果位数加倍
3. 尾数除法无法除尽
4. 规格化左移后补位

#### 常用舍入方法

| 方法 | 规则 | 特点 |
|------|------|------|
| **截断法** | 直接丢弃舍弃位 | 实现简单，误差大，有偏差 |
| **0舍1入法** | 舍弃位最高位为1则进位 | 类似四舍五入，较均匀 |
| **恒置1法** | 保留位末位恒置1 | 不产生进位溢出 |
| **朝+∞舍入** | 正数进位，负数截断 | 结果≥真值 |
| **朝-∞舍入** | 正数截断，负数进位 | 结果≤真值 |
| **朝0舍入** | 直接截断 | 绝对值变小 |
| **就近舍入** | 四舍六入五成双 | **IEEE 754默认**，无偏差 |

#### IEEE 754 舍入模式
1. **就近舍入**（默认）：四舍六入五成双
2. **朝+∞舍入**
3. **朝-∞舍入**
4. **朝0舍入**

#### 0舍1入法示例
- $0.1101|0110 \rightarrow 0.1101$（最高舍弃位为0，舍）
- $0.1101|1011 \rightarrow 0.1110$（最高舍弃位为1，入）

## 四、存储器

### 4.1 存储器分类
- **按介质**：半导体/磁表面/光存储
- **按存取方式**：随机存取(RAM)/顺序存取/直接存取
- **按作用**：主存/Cache/辅存

### 4.2 ⚠️ SRAM vs DRAM

| 特性 | SRAM | DRAM |
|------|------|------|
| 存储元件 | 6管触发器 | 1管1电容 |
| 刷新 | 不需要 | 需要周期性刷新 |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 成本 | 高 | 低 |
| 应用 | Cache | 主存 |

#### DRAM刷新方式
1. **集中刷新**：有"死区"时间
2. **分散刷新**：无死区，但延长存取周期
3. **异步刷新**：折中方案

### 4.3 ⚠️⚠️⚠️ 主存扩展（画图重点）
- **位扩展**：增加数据位宽
- **字扩展**：增加存储容量（需译码器选片）
- **字位同时扩展**：先位扩展后字扩展

#### 连接要点
- 地址线分配：片内地址 + 片选地址
- 控制信号：$\overline{CS}$、$\overline{WE}$、$\overline{OE}$
- 译码器74LS138的使用

### 4.4 ⚠️⚠️ Cache（高频考点）

#### 性能公式
- **命中率**：$h = \frac{N_c}{N_c + N_m}$
- **平均访问时间**：$t_a = h \cdot t_c + (1-h) \cdot t_m$
- **访问效率**：$e = \frac{t_c}{t_a} = \frac{1}{h + (1-h) \cdot r}$，其中 $r = t_m/t_c$

#### 地址映射方式

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 直接映射 | 冲突率高，实现简单 | 大容量Cache |
| 全相联映射 | 冲突率低，硬件复杂 | 小容量Cache |
| 组相联映射 | 折中方案 | 通用 |

#### 替换算法
- **随机替换**、**FIFO**、**LFU**、**LRU**

#### 写策略
- Write Hit：Write-back（写回）/ Write-through（写直达）
- Write Miss：Write-allocate / Non-write-allocate

### 4.5 ⚠️ 虚拟存储器

#### 页式虚存核心概念
- **虚拟地址**：虚拟页号 + 页内偏移
- **物理地址**：物理页号 + 页内偏移
- **页表**：虚拟页 → 物理页映射

#### TLB（快表）
- 缓存常用页表项，位于CPU内部
- **TLB hit → 页必定命中**
- **缺页 → Cache必无对应块**

#### ⚠️ 访存流程（重要）
```
虚拟地址 → 查TLB → (miss)查页表 → (缺页)缺页中断
                ↓ hit
           物理地址 → 查Cache → (miss)访主存
                          ↓ hit
                        返回数据
```

### 4.6 硬盘存储器

#### ⚠️ 关键公式
- **存储容量** = 记录面数 × 柱面数 × 每道容量
- **数据传输率** = 转速(rps) × 每道容量
- **平均等待时间** = 1/(2×转速)
- **平均访问时间** = 寻道时间 + 等待时间 + 传输时间

## 五、指令系统

### 5.1 指令格式
- **基本格式**：操作码(OP) + 地址码(A)
- **指令字长**：定长（RISC）/ 变长（CISC）

### 5.2 地址码分类
- **三地址指令**：OP + A1 + A2 + A3
- **二地址指令**：RR型、RS型、SS型
- **一地址指令**：单操作数 / ACC隐含
- **零地址指令**：堆栈运算 / 无操作数指令

### 5.3 ⚠️ 扩展操作码
- **原理**：操作码长度可变，类似子网划分
- **设计原则**：短码不能是长码前缀

### 5.4 ⚠️⚠️ 寻址方式（核心）

| 寻址方式 | 有效地址EA | 特点 |
|----------|-----------|------|
| 立即寻址 | 操作数=imm | 最快，范围小 |
| 直接寻址 | EA=D | 简单 |
| 寄存器寻址 | EA=Ri | 快速 |
| 间接寻址 | EA=(A) | 范围大 |
| 相对寻址 | EA=(PC)+A | 用于转移指令 |
| 变址寻址 | EA=(Rx)+A | 数组访问 |
| 基址寻址 | EA=(Rb)+A | 程序浮动 |
| 堆栈寻址 | EA=(SP) | LIFO操作 |

### 5.5 CISC vs RISC

| 特性 | CISC | RISC |
|------|------|------|
| 指令数量 | 多 | 少 |
| 指令长度 | 变长 | 定长 |
| 寻址方式 | 多种 | 少（Load/Store） |
| 通用寄存器 | 少 | 多 |
| 流水线 | 难实现 | 易实现 |
| 代表 | x86 | ARM, MIPS |

## 六、中央处理器（CPU）

### 6.1 CPU基本组成
- **运算器**：ALU、通用寄存器、PSW
- **控制器**：PC、IR、CU、时序发生器

#### 寄存器分类
- **用户可见**：通用寄存器、PSW
- **用户不可见**：PC、IR、MAR、MDR

### 6.2 ⚠️ 指令周期

#### 典型划分
1. **取指周期**：PC→MAR→MEM→MDR→IR
2. **间址周期**：取操作数地址
3. **执行周期**：执行具体操作
4. **中断周期**：保存断点，跳转中断服务程序

### 6.3 ⚠️⚠️⚠️ 数据通路分析（MIPS32）

#### 典型指令数据通路
- **lw rt, imm(rs)**
  - 取指：PC→MAR→MEM→MDR→IR；PC+4→PC
  - 计算：R(rs)+imm→ALU→Z
  - 执行：Z→MAR→MEM→MDR→R(rt)

- **sw rt, imm(rs)**
  - 执行：Z→MAR；R(rt)→MDR→MEM

- **add rd, rs, rt**
  - 执行：R(rs)→X；R(rt)+X→ALU→Z→R(rd)

### 6.4 时序控制
- **三级时序**：指令周期 → 机器周期 → 时钟周期
- **时序发生器**：Moore型FSM实现

### 6.5 ⚠️⚠️⚠️ 微程序控制器（重点）

#### 核心概念定义

| 概念 | 定义 |
|------|------|
| **微操作** | 计算机中**最基本、不可再分**的操作，是控制信号的执行结果（如 `PC→MAR`、`(ACC)+1→ACC`） |
| **微命令** | 控制部件向执行部件发出的、用于**控制微操作执行**的命令信号（如使能信号、选通信号） |
| **微指令** | 在**一个时钟周期**内**同时发出**的一组微命令的集合，存储在控制存储器中 |
| **微程序** | 由若干条**微指令**组成的**有序序列**，用于实现一条机器指令的功能 |
| **控制存储器(CM)** | 用于存放全部微程序的**只读存储器(ROM)**，又称μPM |
| **微周期** | 从CM中读取并执行一条微指令所需的时间，通常等于一个时钟周期 |

#### 层次关系
```
机器指令 (用户可见)
    ↓ 一条机器指令对应一段微程序
微程序 (存储在CM中)
    ↓ 一段微程序由多条微指令组成
微指令 (对应一个时钟周期)
    ↓ 一条微指令包含多个微命令
微命令/微操作 (最基本操作，如PC→MAR)
```

#### 微程序控制器组成
1. **控制存储器（CM/μPM）**：ROM实现，存放所有微程序
2. **微指令寄存器（μIR/CMIR）**：存放当前执行的微指令
3. **微地址寄存器（μPC/CMAR）**：存放下一条微指令地址
4. **地址转移逻辑**：形成下一条微指令地址

#### 微指令格式
```
┌─────────────────────┬─────────────────────┐
│    操作控制字段      │    顺序控制字段      │
│  (产生微命令/控制信号) │ (下地址/转移方式)   │
└─────────────────────┴─────────────────────┘
```

#### ⚠️ 操作控制字段编码方式

| 编码方式 | 原理 | 字长 | 速度 | 并行性 |
|---------|------|------|------|--------|
| **直接编码** | 每位对应一个微命令 | 最长 | 最快 | 最强 |
| **字段直接编码** | 互斥微命令分组，组内编码 | 较短 | 较快 | 较强 |
| **字段间接编码** | 字段含义依赖其他字段 | 最短 | 较慢 | 较弱 |
| **混合编码** | 常用命令直接编码+其他字段编码 | 适中 | 适中 | 适中 |

**字段位数计算**：若某字段有n个互斥微命令，需 $\lceil \log_2(n+1) \rceil$ 位

#### 微地址形成方式

| 方式 | 特点 | 应用场景 |
|------|------|----------|
| **计数器方式** | μPC+1，顺序执行 | 微程序顺序段 |
| **下地址字段** | 微指令中直接给出 | 分支/调用 |
| **条件转移** | 根据状态条件选择 | 条件判断 |
| **测试网络** | 操作码直接映射入口 | 取指后进入微程序 |

#### 硬布线 vs 微程序控制器

| 特性 | 硬布线控制器 | 微程序控制器 |
|------|------------|-------------|
| 实现方式 | 组合逻辑电路 | 存储程序（ROM） |
| 速度 | **快** | 较慢（需访问CM） |
| 灵活性 | 差，修改需重新设计 | **好**，修改微程序即可 |
| 规整性 | 不规整 | **规整**，易于设计 |
| 适用场景 | **RISC** | **CISC** |
| 扩展性 | 差 | 好 |

#### 控存容量计算
$$控存容量 = 微指令条数 \times 微指令字长$$

## 七、总线

### 7.1 总线分类
- **数据总线**：双向，宽度=机器字长
- **地址总线**：单向，宽度由存储容量决定
- **控制总线**：传输控制信号

### 7.2 总线仲裁

| 方式    | 特点            |
| ----- | ------------- |
| 链式查询  | 3根线，优先级固定，易饥饿 |
| 计数器查询 | 优先级可变，线数多     |
| 独立请求  | 最快，硬件开销最大     |

### 7.3 ⚠️ 总线通信方式
1. **同步通信**：统一时钟，按最慢设计
2. **异步通信**：握手信号（不互锁/半互锁/全互锁）
3. **半同步通信**：同步+等待状态
4. **分离式通信**：总线操作分离

## 八、输入输出系统

### 8.1 I/O接口
- **作用**：实现主机与外设的信息转换和协调
- **编址**：统一编址 / 独立编址

### 8.2 ⚠️⚠️⚠️ I/O控制方式

#### 程序查询
- CPU轮询设备状态，效率低

#### 程序中断

##### 中断处理流程
1. **中断请求**：设备发出INTR信号
2. **中断判优**：硬件/软件优先级排队
3. **中断响应**：关中断、保护断点、获取入口地址
4. **中断服务**：保护现场→处理→恢复现场
5. **中断返回**：恢复断点，开中断

##### ⚠️ 多重中断条件
- 保护现场后**开中断**
- 恢复现场前**关中断**
- **屏蔽字**控制嵌套：高优先级可抢占低优先级

#### DMA（直接存储器访问）
- **特点**：不经CPU，主存与I/O直接交换
- **与中断对比**：
  - DMA在总线周期结束时响应
  - 中断在指令周期结束时响应

### 8.3 ⚠️ 显示器计算

#### 核心公式
- **显存容量** = 分辨率 × 颜色深度 / 8
- **行频** = 刷新率 × 总行数 × 回扫系数
- **视频带宽** = 分辨率 × 刷新率 × 回扫系数²
