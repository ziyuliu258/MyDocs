# 操作系统概念：第三章 进程 (Processes) 知识总结

## 1. 进程概念 (Process Concept)

### 1.1 进程的定义

- **基本概念**：进程是执行中的程序，是所有计算的基础。
- **程序与进程的区别**：
  - **程序 (Program)**：是被动实体（Passive Entity），如存储在磁盘上的可执行文件，是静态的、永久的。
  - **进程 (Process)**：是主动实体（Active Entity），具有程序计数器（PC）和相关资源集合，是动态的、暂时的。
- **对应关系**：一个程序通过多次执行可对应多个进程；一个进程通过调用关系可对应多个程序。

### 1.2 进程的内存结构
![](Pasted%20image%2020251223233528.png)
进程在内存中的布局通常包含以下部分：

- **文本段 (Text Section)**：存放可执行代码。
- **数据段 (Data Section)**：存放全局变量（包括初始化的和未初始化的）。
- **堆 (Heap)**：用于程序运行时动态分配的内存，**程序员**负责分配释放，分配方式类似链表。
- **栈 (Stack)**：**操作系统**自动管理，用于存放临时数据（如函数参数、返回地址、局部变量）。

### 1.3 进程的状态 (Process State)

进程在执行过程中会发生状态改变，主要包括以下五种状态：
- **新建 (New)**：进程正在被创建。
- **运行 (Running)**：指令正在被执行。
- **等待 (Waiting)**：进程等待某个事件发生（如 I/O 完成或接收信号）。
- **就绪 (Ready)**：进程等待被分配处理器。
- **终止 (Terminated)**：进程完成执行。
### 1.4 进程控制块 (PCB)

操作系统通过 **PCB (Process Control Block)** 来管理进程，它包含与特定进程相关的所有信息：

- **进程状态 (Process state)**
- **程序计数器 (Program counter)**
- **CPU 寄存器 (CPU registers)**
- **CPU 调度信息 (Scheduling information)**：如优先级、调度队列指针等。
- **内存管理信息 (Memory-management information)**：如页表或段表。
- **记账信息 (Accounting information)**：如 CPU 使用时间、时间限制等。
- **I/O 状态信息 (I/O status information)**：如分配给进程的 I/O 设备列表、打开的文件列表。

> **Linux 中的 PCB**：在 Linux 中，PCB 由 `task_struct` 结构体表示，系统通过任务数组（Task array）来保存指向这些结构体的指针。

## 2. 进程调度 (Process Scheduling)

### 2.1 调度队列

- **作业队列 (Job queue)**：系统中所有进程的集合。
- **就绪队列 (Ready queue)**：驻留在内存中、就绪并等待执行的所有进程。
- **设备队列 (Device queues)**：等待特定 I/O 设备的进程集合。
- 进程会根据状态变化在这些队列之间迁移。

### 2.2 调度程序 (Schedulers)

- **长期调度程序 (Long-term scheduler / Job scheduler)**：
  - 选择哪些进程应被调入就绪队列（内存）。
  - 执行频率较低（秒或分钟级），速度可以较慢。
  - 控制 **多道程序设计的程度 (Degree of multiprogramming)**。
  - 负责控制 I/O 繁忙型 (I/O-bound) 和 CPU 繁忙型 (CPU-bound) 进程的混合比例，以优化系统性能。
- **短期调度程序 (Short-term scheduler / CPU scheduler)**：
  - 从就绪队列中选择下一个执行的进程并分配 CPU。
  - 执行频率极高（毫秒级），必须非常快。
- **中期调度程序 (Medium-term scheduler)**：
  - 涉及 **交换 (Swapping)**：将部分执行的进程从内存移出（Swap out）到外存，稍后再移入（Swap in），以减轻内存压力。

### 2.3 上下文切换 (Context Switch)

- **定义**：当 CPU 切换到另一个进程时，系统必须保存当前进程的状态（Context），并加载新进程的保存状态。
- **开销**：上下文切换时间是纯粹的系统开销，期间系统不执行有用的用户工作。
- **速度**：依赖于硬件支持（如寄存器组的数量），典型速度为 1-1000 毫秒。

## 3. 进程操作 (Operations on Processes)

### 3.1 进程创建

- **机制**：父进程创建子进程，形成进程树。
- **资源共享方式**：
  - 父子进程共享所有资源。
  - 子进程共享父进程资源的子集。
  - 父子进程不共享资源（Windows 模式）。
- **执行方式**：
  - 父子进程并发执行。
  - 父进程等待子进程终止。
- **地址空间**：
  - 子进程复制父进程的地址空间（Unix 模式）。
  - 子进程加载一个新程序（Windows 模式）。

#### Unix/Linux 进程创建实例

- **`fork()` 系统调用**：创建新进程。
  - 在父进程中返回子进程的 PID (>0)。
  - 在子进程中返回 0。
  - 若返回负数则表示创建失败。
- **`exec()` 系统调用**：通常在 `fork()` 之后使用，用新程序替换进程的内存空间。
- **`wait()` 系统调用**：父进程阻塞自己，等待子进程完成。

### 3.2 进程终止

- 进程执行完最后一条语句，通过调用 `exit()` 请求操作系统删除它。
- **级联终止 (Cascading termination)**：如果父进程终止，某些操作系统不允许子进程继续存在，所有子进程也会被递归终止。
- **Abort**：父进程可以终止子进程，原因可能包括：子进程使用了超量资源、分配给子进程的任务不再需要、或者父进程自身正在退出。

### 3.3 进程阻塞与挂起 (Blocking & Suspend)

- **阻塞 (Blocking)**：进程因等待事件（如 I/O、信号）而从运行态转为阻塞态（等待态）。
- **挂起 (Suspend)**：
  - 通常是为了调节负载、检查问题或父进程的需求。
  - 挂起通常意味着进程被 **换出到外存**。
  - **区别**：阻塞是被动暂停（仍在内存中），挂起是主动暂停（通常被换出到外存）。

## 4. 进程间通信 (Interprocess Communication, IPC)

协作进程需要 IPC 机制来交换数据。主要有两种模型：

### 4.1 共享内存 (Shared Memory)

- **机制**：建立一块通信进程共享的内存区域。
- **特点**：
  - **速度快**（直接内存访问，无系统调用开销）。
  - 数据的形式和位置由进程决定，不受 OS 控制。
  - **缺点**：程序员必须负责所有的同步工作，复杂度高，容易出错。

### 4.2 消息传递 (Message Passing)

- **机制**：进程通过 OS 提供的机制交换消息，无需共享变量。
- **基本操作**：`send(message)` 和 `receive(message)`。
- **特点**：安全性高，易于同步，但由于涉及系统调用和数据复制，性能开销较大。

#### 消息传递的实现细节

1. **通信链路 (Communication Link)**：
   - **直接通信**：必须显式命名通信对象，如 `send(P, message)`。每对进程间只有一个链路。
   - **间接通信**：通过邮箱（Mailboxes/Ports）发送接收。支持多对多通信，更灵活。
2. **同步 (Synchronization)**：
   - **阻塞 (Blocking)**：视为 **同步**。发送方发送后阻塞直到消息被接收；接收方阻塞直到有消息可用。
   - **非阻塞 (Non-blocking)**：视为 **异步**。发送方发送后立即返回；接收方接收有效消息或空值。
3. **缓冲 (Buffering)**：
   - **零容量**：发送方必须等待接收方（交会）。
   - **有限容量**：队列满时发送方需等待。
   - **无限容量**：发送方从不等待。

### 4.3 经典问题：生产者-消费者问题(Producer-Consumer Problem)
>生产者-消费者问题是协作进程中的一种经典范式 。在这个模型中，系统包含两类进程 ：
>- 生产者 (Producer)：负责生产信息并将其放入缓冲区；
>- 消费者 (Consumer)：负责从缓冲区中取出并消费这些信息。
>为了解决这个问题引入Unbounded-buffer 和 Bounded-buffer。无界的缓冲区生产者可以不停生产；有界缓冲区大小固定，如果buffer满了，生产者必须等待；如果buffer空了，。消费者必须等待。
- 使用 **有界缓冲区 (Bounded-buffer)** 解决。
- 通过共享内存实现，利用 `in` (指向下一个空闲位置) 和 `out` (指向下一个满位置) 指针管理的循环数组。
- 缓冲区满：`(in + 1) % SIZE == out`；缓冲区空：`in == out`。

## 5. 客户机-服务器系统通信

### 5.1 套接字 (Sockets)

- **定义**：通信的端点。
- **构成**：IP 地址与端口号的连接（Concatenation），例如 `161.25.19.8:1625`。
- **机制**：通信由一对套接字组成。

### 5.2 远程过程调用 (RPC)

- **定义**：抽象了网络系统间进程的过程调用机制。
- **存根 (Stubs)**：
  - **客户端存根**：作为服务器上实际过程的代理，负责参数的 **编组 (Marshalling)** 和定位服务器。
  - **服务器存根**：接收消息，解包参数，执行过程，并将结果打包返回。

### 5.3 远程方法调用 (RMI)

- **定义**：Java 的机制，允许一个机器上的 Java 程序调用远程对象的方。
- 类似于 RPC，但是面向对象的。

## 6. 重点习题解析 (来源于课件)

1. **进程创建步骤**：创建进程 **不包括** “由 CPU 调度程序为进程调度”。
   - *解析*：进程创建包括申请 PCB、分配资源、初始化 PCB、插入就绪队列。只有进入就绪队列后，调度程序才会介入。
2. **进程唤醒**：进程被唤醒意味着进程从 **等待态 (Blocked/Waiting)** 转换为 **就绪态 (Ready)**。
   - *解析*：唤醒是指等待的事件发生了（如 I/O 完成），进程具备了运行条件，但还需要等待 CPU 调度。
3. **状态转换因果关系**：
   - **3 -> 1 (阻塞 -> 调度)**：如果就绪队列非空，一个进程因等待事件（转换3）让出 CPU，会立即引起调度程序选择另一个进程运行（转换1）。
   - **2 -> 1 (超时 -> 调度)**：时间片用完（转换2）必然引起调度（转换1）。
   - **3 -> 2 (阻塞 -> 就绪)**：这是不可能的。进程从运行态转为阻塞态后，必须先等待事件完成进入就绪态，而不能直接退回到就绪态（那是超时的情况）。
   - **4 -> 1 (唤醒 -> 调度)**：如果 CPU 空闲且就绪队列为空，当一个进程被唤醒（转换4）进入就绪队列时，会立即引起调度（转换1）。
4. **阻塞与挂起的区别**：
   - **阻塞**：进程/线程被动暂停执行，阻塞的进程仍处于内存中。
   - **挂起**：用户或系统主动暂停执行，挂起的进程通常被换出到外存（磁盘）中。
