# 多模态大语言模型（MLLM）综述

## 阅读论文

- [MASH-VLM: Mitigating Action-Scene Hallucination in Video-LLMs through Disentangled Spatial-Temporal Representations（CVPR 2025）](https://doi.org/10.48550/arXiv.2503.15871)
- [Mulberry: Empowering MLLM with o1-like Reasoning and Reflection via Collective Monte Carlo Tree Search（NeurIPS 2025 Spotlight）](https://doi.org/10.48550/arXiv.2412.18319)
- [Insight-V: Exploring Long-Chain Visual Reasoning with Multimodal Large Language Models（CVPR 2025 Highlight）](https://doi.org/10.48550/arXiv.2411.14432)

## 背景：从感知对齐到深层推理的挑战

近年来，**多模态大语言模型（Multimodal Large Language Models, MLLMs）**通过将**以ViT为基础改进的视觉编码器（如 CLIP, SigLIP）**与**大语言模型（如 Llama, Vicuna, Qwen）**相结合，在理解视觉世界和遵循人类指令方面取得了显著进展 。早期的工作主要集中在**特征对齐（Alignment）**和**指令微调（Instruction Tuning）**上，使得模型能够完成基本的图像描述和简单问答任务 。

> 记：特征对齐可以理解为对多模态的数据形式统一化，转换到LLM能够理解的特征空间。以ViT为代表的将注意力引入图像处理的成果就有助于图像数据向词向量的对齐。

然而，随着应用场景的深入，现有的 MLLM 暴露出了显著的局限性。当前的范式主要停留在“直觉式”的快速感知层面，即基于直观的**模式匹配**生成答案，而在需要深思熟虑的**思考**能力——包括复杂的**时空**逻辑推理、**长链**推导以及精准的**动态场景理解**——方面表现出明显的不足。

结合所读论文，我们可以提炼出 MLLM 目前面临的以下三个核心问题：

### 问题一：缺乏深层逻辑推理与反思能力

尽管 MLLM 在感知任务上表现出色，但在处理需要多步推导的复杂任务（如几何证明、复杂图表分析、科学推理）时，往往显得力不从心。

- **缺失中间推理步骤（CoT）**：现有的 MLLM 通常试图一步到位直接生成答案，缺乏像人类或传统语言大模型（如 `OpenAI o1` ）那样的**思维链（CoT）**过程。这种“黑盒”式的生成方式导致模型无法将复杂问题分解为可管理的子步骤，极易在逻辑链条的中间环节断裂。
- **缺乏自我修正与反思机制**：现有的推理方法通常是静态的单次生成，模型缺乏在推理过程中进行**“反思”（Reflection）**和**“错误定位”**的能力（Mulberry）。一旦早期的推理步骤出错，错误就会在后续步骤中累积，导致最终结果偏离。
- **长链推理的鲁棒性差**：即便通过提示词引导模型进行长链推理，现有的 MLLM 也难以维持长时间的逻辑一致性。简单的**有监督指令微调（SFT）**不足以让模型真正学会处理极长的推理路径，模型往往会在长文本生成中产生幻觉或逻辑跳跃。

### 问题二：动态场景中的时空特征纠缠与幻觉

在视频理解领域，MLLM 面临着比静态图像更严峻的挑战，即如何处理时间（Temporal）与空间（Spatial）信息的交互。

- **动作-场景幻觉（Action-Scene Hallucination）**：MASH-VLM 揭示了一种视频 MLLM 特有的幻觉现象。模型倾向于根据场景上下文错误地推断动作，或者根据动作错误地推断场景。例如，看到“雪山”背景就错误地幻觉出“有人滑雪”，即使画面中并没有人；或者看到“拳击动作”就断定背景是“拳击台”，而忽略了给出的照片背景是图书馆。
- **特征解耦失败**：这种幻觉的根本原因在于模型内部未能有效解耦空间和时间特征。在**注意力机制**运作时，空间和时间 Token 相互交织，导致模型学习到了动作与场景之间的虚假相关性（Spurious Correlations） 。
- **位置编码的偏差**：现有的**RoPE（旋转位置嵌入）**在处理多模态数据时存在缺陷。由于空间 Token 在序列顺序上通常更靠近文本 Token，导致模型在注意力分配上对空间信息产生不合理的偏好，从而忽略了时间动态信息 。

### 问题三：高质量推理数据的稀缺与训练范式的局限

要解决上述推理和幻觉问题，现有的数据和训练流程并不能很好满足改进需求。

- **缺乏过程导向的数据（Process-Oriented Data）**：现有的多模态数据集大多由简单的 `<图像，问题，答案>` 三元组构成，缺乏详细的、显式的、结构化的中间推理步骤。这导致模型只能学习到“是什么”，而学不会“为什么”和“怎么做”。
- **训练范式的单一性**：仅依靠传统的SFT已遭遇瓶颈。Mulberry 和 Insight-V 的研究均表明，为了获得更强的推理能力，必须引入更复杂的训练机制。

综上所述，当前 MLLM 领域正处于从“以感知为主”向“以认知推理为主”转型的阶段。目前的主要痛点集中在：**1) 缺乏类似 o1 的系统性思维链与反思能力；2) 在视频等动态模态中难以解耦时空特征，导致严重的上下文依赖型幻觉；3) 缺乏能够支撑长链推理训练的高质量过程数据与反馈驱动的训练管线。** 接下来的研究重点，如上述三篇论文所示，将聚焦于通过解耦表征（MASH-VLM）、树搜索与反思（Mulberry）以及迭代式的多智能体微调（Insight-V）来突破这些瓶颈。

## 主题概述

### 关键术语定义与技术基础

MLLM的核心在于多模态对齐与融合。主要技术组件包括：

- **视觉编码器**：负责提取图像或视频特征，常用ViT、CLIP或SigLIP等预训练模型。
- **模态投影模块**：将视觉特征映射到语言嵌入空间，常采用线性层、MLP或Q-Former。
- **大语言模型骨干**：如LLaMA-3、Qwen2、Mistral等，提供强大的语言理解与生成能力。
- **训练范式**：通常分为**多模态持续预训练（CPT）**、**指令微调（SFT）**、**强化学习对齐（RLHF/RLAIF/DPO）**等阶段。

典型架构分为**单流**（端到端，如LLaVA-NeXT）和**双流**（解耦视觉与语言，如早期MiniGPT-4）两种。近年来主流研究方向为**高分辨率处理**、**动态token压缩**与**长上下文扩展**。

另外在本文中涉及到的一些关键术语也作简单解释：

- **特征对齐 (Alignment)**：指将不同模态（如视觉与文本）的特征映射到统一的表征空间 。在 MLLM 中，这通常通过桥接层（Projector）将视觉特征转化为语言模型可处理的词 Token 实现 。
- **指令微调 (Instruction Tuning)**：在预训练模型基础上，使用“指令-输入-输出”格式的数据进行训练，旨在教导模型理解人类意图并完成特定任务（如图像描述或视觉问答） 。
- **思维链 (Chain-of-Thought, CoT)**：一种推理诱导技术，要求模型在输出最终答案前，生成一系列中间推理步骤 。它将复杂任务分解为可解释的逻辑链条 。
- **蒙特卡洛树搜索 (Monte Carlo Tree Search, MCTS)**：一种通过在决策树中进行启发式探索来寻找最优路径的方法 。在推理任务中，它被用于搜索和定位通往正确答案的最佳逻辑步骤 。

### 技术发展脉络

MLLM发展可大致分为三个阶段：
（1）探索阶段（2022—2023年初）：Flamingo、BLIP-2、Kosmos-1等开创性工作证明了冻结LLM+轻量投影的可行性，LLaVA通过指令数据实现开源突破。
（2）快速发展阶段（2023—2024）：GPT-4V、LLaVA-1.5、CogVLM、Qwen-VL等模型显著提升性能，高分辨率处理（如LLaVA-NeXT）、视频扩展（如Video-LLaVA）成为热点。
（3）成熟优化阶段（2024—2025）：InternVL3.5、Qwen2-VL、LLaVA-OneVision等模型聚焦长链推理、视频理解、效率优化与可靠性提升，开源与闭源模型差距大幅缩小。同时，专用基准（如Video-MME、LongVideoBench）与幻觉缓解方法（如时空解耦）层出不穷。

接下来将对以上提到的三篇论文进行深度梳理，并针对它们在解决 MLLM 痛点时的不同路径进行对比分析。

## 文献综述

### 核心论文梳理

#### **MASH-VLM：解决感知层面的动作-场景幻觉**

如今视频大语言模型中普遍存在“动作-场景幻觉”问题，即模型根据场景直觉猜测动作，或根据动作推测场景。 文章指出，这是因为现有的模型在注意力机制中将空间和时间特征混为一谈，且位置编码（RoPE）倾向于让文本 Token 更多地关注空间信息 。

##### 技术创新

- **Harmonic-RoPE**：分配额外的平衡位置 ID，确保具有相同 ID 的空间标记和时间标记与文本标记之间保持相等的相对位置距离 。
- **DST-Attention**：通过使用掩码注意力来解耦这些标记，防止空间标记与时间标记之间发生直接交互 。
- **贡献**：提出了 **UNSCENE 基准测试**，专门评估模型在非典型场景（如论文首张图中在图书馆拳击）下的幻觉表现 。

###### 输入处理

给定一个包含 T 个帧的视频 $V∈R^{T×C×H×W}$，我们首先通过预训练的视觉编码器提取每一帧的特征。为了减少标记（token）数量并增强多模态理解，我们采用了空间池化和时间池化。

具体而言，视觉特征首先通过步长为 2 的投影层进行聚合，以获得空间标记 $X_{spat}∈R^T×2^H×2^{W×D}$。随后，为了捕捉时间动态，我们将空间标记沿时间维度进行池化，得到时间标记 $X_{temp}∈R^{T×D}$。最后，我们将这些标记拼接，形成输入序列：
$$
X=[X_{spat},X_{temp},X_{text}]
$$
其中 $X_{text}$ 是文本查询的嵌入。该序列被输入到集成了**平衡位置编码组件**和**解耦组件**的LLM中。

###### Harmonic-RoPE：平衡位置编码

传统的旋转位置编码（RoPE）由于采用一维顺序编码，会导致文本 Token 倾向于过度关注与其序列位置更接近的视觉 Token（通常是空间 Token），产生幻觉偏差 8888。MASH-VLM 提出了 **Harmonic-RoPE**，通过扩展位置 ID 的维度来实现平衡 。

其核心在于将位置 ID 分解为**不同（Distinct）** 和**平衡（Balanced）** 两部分 ：

- 不同位置 ID ($p_{distinct}$)：维持传统的线性顺序编码：
  $$p_{distinct} = [1, 2, \dots, N+M+L] $$

- 平衡位置 ID ($p_{balanced}$)：通过偏移让空间和时间 Token 相对文本 Token 具有相等的距离：

  $$\begin{aligned} p_{temporal} &= [N-M+1, \dots, N] \\ p_{spatial} &= [1, 2, \dots, N] \\ p_{text} &= [N+1, \dots, N+L] \end{aligned} $$

- 谐波 ID 选择：在特征向量的偶数维度对采用平衡 ID，在奇数维度对采用不同 ID，确保模型既能区分顺序又能平衡时空权重 ：

  $$p_{harmonic}(k) = \begin{cases} p_{balanced}(k), & \text{if } k \text{ is even} \\ p_{distinct}(k), & \text{if } k \text{ is odd} \end{cases}$$

###### DST-Attention：解耦时空注意力机制

为了防止空间和时间特征在 LLM 的多层注意力计算中重新交织，MASH-VLM 设计了 **DST-Attention** 。该机制通过一个特殊的掩码 $M^{DST}$ 限制 Token 间的交互：
- 解耦掩码 ($M^D$)：禁止空间 Token ($i$) 与时间 Token ($j$) 之间的直接互视：

  $$M^D(i, j) = \begin{cases} -\infty, & \text{if } \text{type}(i) \neq \text{type}(j) \\ 0, & \text{otherwise} \end{cases}$$

- **结构化建模 ($M^{ST}$)**：对于**时间 Token** 采用**因果注意力（Causal Attention）以保持序列依赖；对于空间 Token** 采用**双向注意力（Bi-directional Attention）**，因为空间维度天然不具备方向性。

#### Mulberry：基于集体蒙特卡洛树搜索（CoMCTS）的推理增强方法

Mulberry 旨在解决当前多模态大语言模型（MLLMs）在处理复杂任务时因缺乏显式中间推理步骤而导致的“直接预测故障”。其核心方法论是提出了**集体蒙特卡洛树搜索（Collective Monte Carlo Tree Search, CoMCTS）**，通过引入集体学习（Collective Learning）的概念，克服了传统 MCTS 在多模态空间中搜索效率低下及容易陷入同质化低质量节点的问题 。

##### CoMCTS 算法流程

CoMCTS 利用多个模型 $\{\pi_1, \pi_2, \dots, \pi_K\}$ 的集体知识来共同构建推理树。每个搜索迭代包含以下四个关键操作：
- **(a) 联合扩展（Joint Expansion）**：从当前起始节点出发，利用多个 MLLM 并行生成多样化的候选推理路径，直到终止节点。
- (b) 模拟与错误定位（Simulation and Error Positioning）：利用集体知识评估候选节点的潜力值 $\bar{R}(s_i^j)$，并定位错误步骤进行剪枝 ：

  $$\bar{R}(s_i^j) = \frac{1}{K} \sum_{l=1}^K \pi_l(\cdot | prompt_{eval}, Q, Parent(s_i^j), s_i^j) $$

  $$S_{candidate}^* = \{s_i^j \in S_{candidate} | R(s_i^j) \ge t\}$$

  只有得分超过阈值 $t$ 的节点会被保留，这大大减少了无效搜索时间。
- (c) 反向传播（Backpropagation）：从叶节点向上更新推理树中每个节点 $s$ 的统计信息，包括访问计数 $N(s)$ 和节点价值 $V(s)$ ：
  $$
  V(s) \leftarrow \frac{N(s) \cdot V(s) + \sum_{s_l \in Child(s)} R(s_l)}{N(s) + CountChild(S_{candidate}^*, s)}
  $$

  - **$V(s)$**：节点原本的平均价值
  - **$N(s)$**：该节点之前被访问的总次数
  - **$\sum R(s_l)$**：本次新探索产生的子节点的奖励总和

  - **$CountChild$**：新增加的子节点数量
  同时更新该节点的访问热度 ：
  $$
  N(s) \leftarrow N(s) + CountChild(S_{candidate}^*, s)
  $$
  
- (d) 选择（Selection）：基于**上置信界（UCB）**算法选择下一个搜索的起始节点，以平衡**探索**与**利用** ：

  $$m = \arg \max_{s \in S_{candidate}^*} V(s) + c \sqrt{\frac{\log N(\hat{s})}{N(s)}}$$

##### 反思推理路径的构建

为了赋予模型自我修正能力，Mulberry 扩展了 CoMCTS 以搜索反思路径（Reflective Reasoning Paths）。通过识别有效路径 $Y$ 中的节点 $s$ 及其负面兄弟节点 $s_{neg}$，构建包含“错误-反思-修正”转折的轨迹：

$$Y_{reflect} = Replace(Y, s, (s_{neg}, prompt_{reflect}, s))$$

其中 $prompt_{reflect}$ 引导模型进行逻辑重新校准。

##### 集体监督微调（CoSFT）

基于 CoMCTS 构建的 **Mulberry-260k** 数据集（包含 260,000 个推理树），模型通过**集体监督微调（CoSFT**）进行训练。其训练目标包含两个部分：

（1）标准推理损失： 旨在学习集体识别出的有效推理路径。

$$\mathcal{L}_{CoSFT}(\pi_k) = \sum_{(Q, Y) \in D} \log \pi_k(Y | Q)$$

（2）反思增强损失： 训练模型在遇到错误逻辑时能够自主进行步进式反思 。

$$\mathcal{L}_{CoSFT-Re}(\pi_k) = \sum_{(Q, Y_{reflect}) \in D} \log \pi_k(Y_{reflect} | Q)$$

通过这种从集体搜索到集体学习的闭环，Mulberry 显著提升了在 MathVista 和 MMMU 等复杂推理任务中的性能。

#### **Insight-V：基于多智能体系统与迭代偏好优化的长链视觉推理方法**

Insight-V 旨在解决多模态大语言模型（MLLMs）在处理复杂视觉任务时，缺乏高质量长链推理数据和有效训练流水线的问题 。其核心方法论结合了自动化的数据生产管线、推理与摘要解耦的智能体架构，以及增强生成稳定性的**迭代 DPO 算法**。

##### **结构化长链推理数据构建**

为了在无需人工劳动的条件下生产大规模、高质量的推理数据，Insight-V 采用了两步走战略。

（1）渐进式推理生成（Progressive Generation）：

利用推理生成器以 JSON 格式输出结构化推理过程。在每一步中，模型需提供当前步骤摘要、详细响应及后续动作（继续或摘要）。其状态生成过程可表示为：

$$R_t = M(I, Q, [R_1, \dots, R_{t-1}], A)$$

其中 $R_t$ 为第 $t$ 步的响应，$I$ 为图像，$Q$ 为问题，$A$ 为上一步确定的动作。

（2）多粒度评估系统（Multi-Granularity Assessment）：

生成后的数据需经过强 LLM（如 Qwen2）的初步答案过滤，以及高级 MLLM（如 Qwen2-VL）的路径打分。打分 Agent 会根据推理的步骤准确性和细节程度给出 1-100 的评分，仅保留高分路径用于训练。

##### **推理与摘要双智能体架构设计**

Insight-V 认为将推理与问答混合在单一过程中会增加错误风险，因此提出了多智能体协作框架。

- **推理智能体（Reasoning Agent）：** 专注于为输入查询生成详细的、步骤分明的中间推理过程。
- **摘要智能体（Summary Agent）：** 负责评估并有选择地利用推理结果来得出最终答案。
- **鲁棒性训练：** 为了防止摘要智能体盲目复制可能存在的推理错误，其训练集同时包含了“最优路径”和“有缺陷路径”，促使模型进行批判性评估而非简单聚合。

##### **两阶段训练与迭代偏好优化（Iterative DPO）**

模型的训练分为监督微调（SFT）和对齐优化两个阶段 。在对齐阶段，为了解决传统离线 DPO 随着模型演进效果减弱的问题，Insight-V 引入了迭代 DPO 算法 。

<img src="attachments/image-20251219192629790.png" alt="image-20251219192629790" style="zoom: 50%;" />

（1）偏好模型建模：

采用 Bradley-Terry (BT) 模型来近似人类偏好分布：

$$P(y_1 > y_2 | x) = \sigma(r^*(x, y_1) - r^*(x, y_2))$$

其中 $\sigma$ 为逻辑（Logistic）函数，$r^*$ 为潜在奖励模型。

（2）迭代优化目标：

通过最小化负对数似然函数，使模型生成的推理步骤与人类（或专家模型）的思维过程对齐：

$$\mathcal{L}_{R}(r_{\phi}, \mathcal{D}) = - \mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} [\log \sigma(r_{\phi}(x, y_w) - r_{\phi}(x, y_l))]$$

通过多轮迭代采样与对齐，推理智能体能够生成更具结构化且准确度更高的推理长链。

#### **MASH-VLM、Mulberry 与 Insight-V 的多维对比分析**

为了深入理解当前多模态模型在解决幻觉与增强推理能力方面的技术路径，我们将 **MASH-VLM**、**Mulberry** 与 **Insight-V** 纳入统一的分析框架。这三项工作虽然目标一致（提升模型性能），但其切入点分别位于**模型内部架构**、**推理搜索算法**以及**系统级数据管线**三个截然不同的层面。

##### **1. 核心动机与问题定义的差异**

- **MASH-VLM（表征层面的解纠缠）**：该工作的核心痛点在于**视频理解中的时空混淆**。现有模型常因无法区分静态场景（Scene）与动态动作（Action）而产生幻觉（例如看到雪山背景就错误预测滑雪动作）。因此，MASH-VLM 侧重于通过**内部表征的解耦**来从根源上消除这种相关性偏差。
- **Mulberry（推理层面的深度搜索）**：该工作旨在解决 MLLM 缺乏类似 OpenAI o1 的**系统性思维（System 2 Thinking）的问题。它认为现有模型倾向于“直接预测”，缺乏中间步骤。Mulberry 致力于通过集体树搜索（CoMCTS）**来显式地构建包含“错误-反思-修正”的复杂推理链。
- **Insight-V（数据与系统层面的扩展）**：该工作主要解决**高质量长链推理数据的稀缺性**与训练管线的缺失。与 Mulberry 关注搜索算法不同，Insight-V 更侧重于构建一个可扩展的**自动化数据工厂**（渐进式生成+多粒度评估），并通过**多智能体协作**（推理与摘要分离）来规避长链推理中的误差累积。

##### **2. 方法论范式的演进对比**

| **维度**          | **MASH-VLM**                                                | **Mulberry**                                                 | **Insight-V**                                                |
| ----------------- | ----------------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **技术范式**      | **架构修正 (Architectural Mod.)**                           | **推理搜索 (Test-time Search)**                              | **系统工程 (System Engineering)**                            |
| **核心机制**      | **DST-Attention** (注意力掩码) **Harmonic-RoPE** (位置编码) | **CoMCTS** (集体蒙特卡洛树搜索) **Collective Knowledge** (集体智慧) | **Multi-Agent** (推理/摘要双智能体)<br />**Iterative DPO** (迭代偏好优化) |
| **幻觉/错误处理** | **预防机制**：通过物理隔离时空特征，防止错误关联的产生。    | **反思机制**：允许模型犯错，通过搜索负面节点并学习“反思路径”进行自我修正。 | **过滤机制**：通过摘要智能体（Summary Agent）评估推理过程，过滤掉错误的中间步骤。 |
| **数据策略**      | 依赖常规视频指令微调数据，但在模型内部改变数据交互方式。    | 构建 **Mulberry-260k**：包含推理树和反思路径的复杂数据集。   | 构建流水线生成数据：渐进式生成 + 多粒度打分筛选 。           |

##### 3. 关键技术特性的深度解析

（1）内部干预 vs. 外部引导

MASH-VLM 代表了内部干预的典型路径。它不改变训练数据的内容，而是直接修改 Transformer 的注意力机制（DST-Attention）和位置编码（Harmonic-RoPE），强制模型在特征提取阶段就“看清”事实。

相比之下，Mulberry 和 Insight-V 均属于外部引导。它们不修改 LLM 的底层架构（如 LLaVA 或 Qwen），而是通过外部的算法（MCTS 搜索）或系统设计（双 Agent 协作）来引导模型生成更高质量的输出。

（2）推理路径的构建：树状搜索 vs. 线性迭代

在构建推理链（CoT）时，Mulberry 采用了非线性的树状结构（Tree Search）。它利用集体知识在每一步进行扩展、模拟和回溯，寻找全局最优解，并特意保留“反思”路径。

Insight-V 则采用了更为线性的**渐进式生成（Progressive Generation）**配合迭代优化。它通过“生成-评估-生成”的循环来延长推理链，并利用迭代 DPO（Iterative DPO）来不断对齐人类偏好 。

（3）对“错误”的利用方式
- **MASH-VLM** 视错误（幻觉）为**噪声**，试图通过解耦掩码（Masking）将其屏蔽。
- **Insight-V** 视错误为**废料**，通过摘要智能体（Summary Agent）将其丢弃，只保留有用的推理精华。
- **Mulberry** 视错误为**教材**，它通过 $Y_{reflect}$ 路径显式地训练模型识别错误并进行反思，认为“从错误中学习”是提升推理能力的关键。

## 总结与趋势展望

通过对比可以看出，多模态模型的演进正呈现出**从感知对齐向深度推理跃迁**的趋势。

- 如果任务侧重于**高保真的感知与描述**（如视频监控、场景分析），**MASH-VLM** 这种底层表征解耦的方法更为基础且有效。
- 如果任务侧重于**复杂的逻辑推演与数学解题**，**Mulberry** 这种基于搜索的反思式推理（System 2）代表了未来的方向。
- 如果目标是**工程化落地与数据规模化**，**Insight-V** 提供的自动化管线与多智能体架构提供了最具可操作性的解决方案。

这三者互为补充，未来的理想模型可能会同时具备 MASH-VLM 的解耦感知能力、Mulberry 的深度反思能力以及 Insight-V 的稳健训练框架。
